Planteamiento del Problema

Accidentabilidad en el Sector Industrial

La omisión o el uso incorrecto de los Equipos de Protección Personal (EPP) constituye una de las principales causas de lesiones graves y fatalidades en sectores de alto riesgo como la construcción, minería y manufactura.

Deficiencias de la Supervisión Manual

  • Intermitencia: Imposibilidad de mantener una vigilancia exhaustiva sobre todo el personal simultáneamente.
  • Error humano: Susceptibilidad a la fatiga visual, distracciones o complacencia por parte del supervisor.
  • Costos operativos: Alta inversión requerida para mantener personal dedicado exclusivamente a labores de verificación.

Objetivos del Proyecto

Objetivo General

Desarrollar e implementar un sistema informático capaz de identificar en tiempo real la presencia de equipos de seguridad en el personal, automatizando la decisión de control de acceso a zonas restringidas.


Objetivos Específicos

  • Entrenar un modelo de detección de objetos capaz de reconocer cinco categorías de EPPs bajo diversas condiciones de iluminación y perspectiva.
  • Integrar el modelo predictivo con una interfaz gráfica intuitiva que procese flujos de video continuo.
  • Diseñar una arquitectura escalable que permita procesar la inferencia en la nube sin requerir hardware especializado en el punto de acceso.

Marco Tecnológico Base

Algoritmo YOLO11 (You Only Look Once)

YOLO es el estándar industrial actual para la detección de objetos en tiempo real. A diferencia de clasificadores tradicionales, YOLO aplica una única red neuronal a toda la imagen, dividiéndola en una cuadrícula para predecir cuadros delimitadores (bounding boxes) y probabilidades de clase simultáneamente.

Plataforma Roboflow

Plataforma integral de Computer Vision utilizada para la gestión del ciclo de vida del modelo. Se empleó para el preprocesamiento de imágenes, el aumento de datos (Data Augmentation) y el despliegue del modelo entrenado mediante una Interfaz de Programación de Aplicaciones (API) REST.

Proceso de Entrenamiento y Datos

El modelo fue entrenado a partir de un conjunto de datos (dataset) compuesto por miles de imágenes recolectadas y anotadas manualmente. La red fue instruida para clasificar y localizar las siguientes categorías requeridas para la validación de acceso:

Clase (Modelo) Descripción del Objeto Criticidad para Acceso
botasCalzado / Botas de seguridadRequerido
cascoCasco de protecciónRequerido
chalecoChaleco de alta visibilidadRequerido
mascaraMáscara de protecciónRequerido
orejerasOrejeras de protecciónRequerido

Nota: La clase 'persona' se emplea como referencia para el tracking general. Otras clases como 't_quimico' son ignoradas por el sistema.

Arquitectura del Sistema en Tiempo Real

Flujo de Datos

  1. Captura: Inicialización del hardware de video (Webcam) a través de los protocolos del navegador.
  2. Extracción: Muestreo de fotogramas e inyección en formato Base64 hacia el servidor de inferencia.
  3. Predicción: La API de Roboflow evalúa la matriz y retorna un documento JSON con coordenadas espaciales y niveles de confianza.
  4. Renderizado: La aplicación cliente dibuja la superposición gráfica sobre el flujo de video original.

Motor de Decisión (Control Lógico)

La autorización se basa en una intersección estricta de conjuntos de datos. El sistema evalúa si el arreglo de clases detectadas contiene la totalidad de los 5 elementos requeridos.

  • Ausencia total: Alerta crítica.
  • Detección parcial: Retroalimentación granular indicando exactamente el elemento faltante.
  • Detección total: Señal de autorización de acceso.
